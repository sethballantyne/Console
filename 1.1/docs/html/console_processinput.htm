Console_ProcessInput

Summary

#include "console.h"

void console::Console_ProcessInput(console::Console& console, SDL_Event* event);

Description

Console_ProcessInput handles all user input when the console is visible; without calling this, the user will type and not see anything being printed in the input buffer. event should be the SDL_Event instance you're using to handle events in your main loop. The console makes use of unicode to retrieve the ASCII values of keys pressed, so unicode must be enabled by calling SDL_EnableUNICODE(1) during your games initialization. Failing to do this will result in keypresses not being detected.

Console_ProcessInput handles the following keys. The explanations are fairly obvious, but are here for completeness.

BACKSPACE moves the cursor one character to the left, deleting the character.
ENTER submits any text written in the input buffer.
PAGE UP scrolls the output buffer upwards.
PAGE DOWN scrolls the output buffer downwards.

Return Value

Console_ProcessInput doesn't return any values.
