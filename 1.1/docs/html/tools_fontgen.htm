4. Fontgen

Summary

fontgen [font=<path>] [output=<path>] [numchars=<num>] [width=<character width>] [height=<character height>] [initial=<first character>]

Description

Fontgen is a program that's designed to make creating built-in fonts a little less time consuming, by taking an existing bitmap font and converting it into a three dimensional C/C++ array that's usable by the console. It will also write several #define's based on the values given to convert the font to source code:
DEFAULT_FONT_HEIGHT
the height of the font in pixels.
DEFAULT_FONT_WIDTH
the pixel width of each character in the font.
DEFAULT_FONT_NUM_GLYPHS
the specified number of characters in the font.
DEFAULT_FONT_FIRST_CHAR
The first character in the font.
DEFAULT_FONT_LAST_CHAR
The last character in the font. This is calculated by fontgen based on the decimal value passed for the initial character, plus the number of characters in the font, minus 1 (fontgen starts counting from the initial character).
The bitmap font specified in the font argument must be laid out in a particular way. Specifically, there must be:
  • A gap of 2 pixels above each character.
  • A gap of 3 pixels seperating each character; this includes the starting character, which must have a 3 pixel gap to its left.
  • All characters must be on the same row.
  • All characters must be ordered by their ASCII values, smallest to highest.
Naturally, the required gaps are not factored into the size of each character and are not present in the resulting array that's generated by fontgen.

Below is an example of how a font should be formatted: the green highlights the 2 pixel gap required at the top, and the magenta highlights the 3 pixel gap required of all characters. The first character in the font is a space character (hence why it's blank), and also has a 3 pixel gap to its left.

Arguments

Every argument that fontgen accepts is optional; when an argument isn't passed via command line, a default value will be used.

font the path (including filename) of the font to convert. The default for this option is to look for default_font.png in the same directory as fontgen. The image can be either BMP, GIF, EXIF, JPG, PNG or TIFF format.
output the path (including filename) of the file to write the generated font to. Note: if a file at the specified path already exists, it'll be automatically overwritten. The default for this option is to write to output.txt in the same directory as fontgen.
numchars the number of characters in the font; the default is 95.
width How wide each character is in pixels; the default is 10.
height how high each character is in pixels; the default is 14.
initial the decimal value of the first character in the font; the default is 32 (ASCII code for the space key).

Example

fontgen font=C:\dir\font.png output=C:\dir\font.txt width=12 height=14 initial=33

This loads the font from C:\dir\font.png. Each character has been configured to have a width of 12 pixels and a height of 14 pixels. The first character in the font is decimal 33 (the '!' character) and the generated source code will be written to C:\dir\font.txt.
