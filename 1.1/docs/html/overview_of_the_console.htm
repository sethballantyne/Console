Overview of the Console

This section describes how the console is broken down. It's mostly intended for people wanting to modify the console, but people only wanting to incorporate it into a project may find it helps get a lay of the land.

The console is broken up in to two sections: the output buffer and the input buffer. As the names suggest, the output buffer is where all the output is printed and internally is represented by the OutputBuffer structure. The input buffer is where the user input appears and is represented by the InputBuffer structure. There's a small pixel gap between both buffers, represented by the define CONSOLE_GAP_BETWEEN_BUFFERS in the console header. Output is rendered from the bottom up, so this gap has a small effect on where the first and subsequent lines of output are placed on the console.

How much text the input buffer can hold is determined when Console_Init is called; at the time of writing this is based on the width of the console, divided by the width of the font being used. The output buffer will hold the same amount of text in each line and if text is printed via Console_Print that exceeds this length, it'll be split up into multiple lines. The input buffer doesn't horizontally scroll, so once you reach the limit that's it; you'll need to hit backspace or enter to clear it. The output buffer will scroll on its own accord once the screen is full, but you can't press a key to scroll it up or down; these are features planned for a future release. The built-in console command cls will clear the output buffer when desired.

As noted in the API section, with the exception of the command-related fuctions, Console_Init must be called before any of the other functions are used. Unless you're modifying the console, the only structure you'll need to worry about is the Console structure. You'll never directly interact with the InputBuffer and OutputBuffer structures.

The console uses a built in font, which you'll find in the defaults header. a tool exists that generates fonts for use by the console if you wish to modify or replace this font. All characters in the font must be of the same width and height; if you change the width or height of the existing font or replace the font entirely, don't forget to update the relevant defines in the defaults header. Hint: use the tool!
