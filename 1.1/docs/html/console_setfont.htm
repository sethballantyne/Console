Console_SetFont

Summary

#include "console.h"

int console::Console_SetFont(Console& console, SDL_Surface* fontSurface, unsigned int numChars, unsigned int charWidth, unsigned int charHeight, unsigned int startingChar, SDL_Colour* cursorColour)

Description

Sets the console font to the bitmap font specified in fontSurface. If fontSurface evaluates to nullptr, the console will fallback to the built-in font; this also means that nullptr can be used to clear the current font if you wish. In this case, the values passed to numChars, charWidth, charHeight, startingChar and cursorColour are disregarded. Note that if the console falls back to the default font, you don't need to create a cursor for it; the console will create one.

Bitmap fonts passed to Console_SetFont must be formatted thusly:
  • The first character begins at pixel coordinate 0,0.
  • Each character is assumed to be charWidth x charHeight in size, with each character directly following the previous; there should be no gaps between characters.
  • Every character must be placed on a single horizontal line.
  • Characters must be ordered by their ASCII values, smallest to largest.
  • If using a colour key, it should be using the same colour specified during initilization; see Console_Init.
An example of a valid bitmap font can be found here. This font contains 95 characters starting from decimal 32 (space) and ending at decimal 126 (the '~' character). Each character is 18x18 pixels in size.

fontSurface the bitmap font the console should use; it's up to the programmer to create this instance of SDL_Surface before calling Console_SetFont and free it when it's no longer needed.
numChars the number of characters in the font.
charWidth the character width of the font, in pixels.
charHeight the character height of the font, in pixels.
startingChar the ASCII value of the first character in the font.
cursorColour the desired colour of the cursor for the new font. If nullptr is passed, the default font colour specified in defaults.h is used.

Return Value

CONSOLE_RET_SUCCESS The console's font was successfully changed to the specified font in fontSurface.
CONSOLE_RET_NULLPTR_ARGUMENT fontSurface evaluated to nullptr, resulting in the console falling back to the built-in font.
