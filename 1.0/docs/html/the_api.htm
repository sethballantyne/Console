The API

This section describes how to incorporate the console into an existing project. Before any functions are called, you must create an instance of the Console structure. Once this is done, you're free to initialize the console, poll for input and render it to the screen or another instance SDL_Surface. The console supports custom commands should be they be needed; these can be created by calling Console_RegisterCommand and are executed by calling Console_ExecuteCommand. The console must be initialised before any other functions can be called, with the exception of the command-related functions Console_IsCommand, Console_RegisterCommand and Console_ExecuteCommand.

You'll only ever need to interact with the Console structure if you're not planning on modifying the source code. There are several other structures within the project that are used internally however, and these have been documented for completeness.
