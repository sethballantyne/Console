Console_Init

Summary

#include "console.h"

int console::Console_Init(console::Console& console, SDL_Surface* screen, SDL_Colour* consoleColour, SDL_Colour* fontColour, SDL_Colour* transparencyColour);

Description

Initializes the console as a software surface; this function also creates the built-in font as an SDL_Surface. During initialization, the console's width will be set to the same as screen, but half its height. If nullptr is passed for any of the colour arguments, a default colour will be used for that argument.

screen The SDL_Surface being used to render to the screen.
consoleColour The desired background colour of the console. If nullptr is passed, black is used.
fontColour The desired colour of each character in the built-in font. If nullptr is passed, white is used.
transparencyColour The colour being used for transparency during blits; this is required for blitting the font. If nullptr is passed, magenta (R=255, G=0, B=255) is used.

Return Value

CONSOLE_RET_SUCCESS The console was successfully initialized.
CONSOLE_RET_CREATE_SURFACE_FAIL SDL_CreateRGBSurface failed to create the consoles surface or the surface for the built-in font. Each surface is created using screen's pixel format along with its width and height, so verify these are valid. Otherwise, check the implementation of Console_Init: it could be the flags passed to SDL_CreateRGBSurface which are SDL_SWSURFACE | SDL_SRCCOLORKEY at the time of writing.
